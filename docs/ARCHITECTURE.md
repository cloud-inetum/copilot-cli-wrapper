# Architecture

## Overview

The wrapper sits **between the user and the native GitHub Copilot CLI**.  It
intercepts all input, routes built-in commands internally, and forwards every
other prompt to the real CLI while capturing and logging the output.

```
┌──────────────┐   prompt / command   ┌─────────────────────────┐
│   User / TTY │ ──────────────────►  │   CopilotCliWrapper      │
└──────────────┘                      │  (orchestrator)          │
                                      └───────────┬─────────────-┘
                                                  │
                      ┌───────────────────────────┼──────────────────┐
                      │                           │                  │
               ┌──────▼──────┐           ┌────────▼───────┐  ┌──────▼──────┐
               │CommandParser│           │  CliExecutor   │  │ LogManager  │
               │  (routing)  │           │ (native CLI)   │  │  (logging)  │
               └─────────────┘           └────────────────┘  └─────────────┘
```

## Component responsibilities

| Component | File | Responsibility |
|-----------|------|----------------|
| `Program` | `src/Program.cs` | CLI argument parsing, wiring, entry point |
| `CopilotCliWrapper` | `src/CopilotCliWrapper.cs` | Session lifecycle, REPL loop, command dispatch |
| `CommandParser` | `src/CommandParser.cs` | Identify and parse wrapper built-ins |
| `CliExecutor` | `src/CliExecutor.cs` | Locate and invoke the native CLI binary |
| `LogManager` | `src/LogManager.cs` | Write session Markdown, produce exports |
| `ConversationEntry` | `src/ConversationEntry.cs` | Immutable Q&A data record |
| `SessionInfo` | `src/Models/SessionInfo.cs` | Mutable session metadata (model, platform…) |
| `ExportFormat` | `src/Models/ExportFormat.cs` | Enum of supported export formats |

## Data flow — interactive mode

```
1. User types input
2. CopilotCliWrapper.RunInteractiveAsync reads line
3. CommandParser.TryParse checks if it is a built-in
   a. Built-in  → handled locally (history, search, export, model, clear, exit)
   b. Otherwise → CliExecutor.RunAsync(arguments)
4. CLI output is streamed to the console in real time
5. Q&A pair is saved to List<ConversationEntry> (memory)
6. LogManager.WriteEntry appends Markdown to the session log file
```

## Data flow — non-interactive mode (`-p`)

```
1. Program.cs parses -p / -m / --cli-path flags
2. CopilotCliWrapper.RunSingleQueryAsync is called
3. Steps 3-6 from interactive mode
4. LogManager.WriteSessionSummary appends summary
5. Process exits
```

## Cross-platform support

`CliExecutor` detects the host OS and chooses the appropriate shell:

| OS | Shell used |
|----|-----------|
| Windows | `cmd.exe /c "copilot.cmd" …` |
| macOS/Linux | `$SHELL -c "copilot …"` (falls back to `/bin/bash`) |

Binary auto-discovery order:

1. `COPILOT_CLI_PATH` environment variable
2. Well-known installation paths (`%APPDATA%\npm\`, `/usr/local/bin/`, etc.)
3. Plain name resolved via system `PATH`

## Log file layout

```
~/.copilot-wrapper/
└── logs/
    ├── session_2026-03-28_14-30-45.md   ← auto-created each session
    ├── export_2026-03-28_15-00-00.json  ← generated by 'export json'
    └── export_2026-03-28_15-01-00.csv
```

## Extension points

* **Custom log directory** — pass a different path to `LogManager(logDirectory)`.
* **Custom CLI path** — set `COPILOT_CLI_PATH` or pass `cliPath` to `CliExecutor`.
* **Additional export formats** — extend `ExportFormat` and add a branch in
  `LogManager.Export`.
* **New wrapper commands** — add entries to `CommandParser.SimpleCommands` and a
  handler case in `CopilotCliWrapper.HandleWrapperCommandAsync`.
